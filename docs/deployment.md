# ClassLapse 部署到希沃白板

> 把一台希沃白板（或任何 Windows 10/11 教室一体机）从"空白系统"变成"开机即静默拍照"的全步骤。预计耗时 15 分钟。

## 0. 准备产物

在你的开发机（Windows 上有 .NET 8 SDK）：

```powershell
git pull
./publish.ps1 -Zip
```

会产出：
- `publish/ClassLapse.exe`（~65 MB，已内嵌 .NET 8 Runtime）
- `publish/README.md` + `publish/deployment.md`
- `ClassLapse-v0.1.0-win-x64.zip`（解压即用）

把 zip 拷到 U 盘，或者放 OneDrive、坚果云之类。

## 1. 拷贝到希沃白板

希沃一体机的 Windows 桌面上：

1. 解压 zip 到 `D:\Tools\ClassLapse\`（**不要**放 `C:\Program Files`，那里需要管理员权限会让首次写配置失败）
2. 双击 `ClassLapse.exe` 启动

## 2. 首次运行向导

启动后会自动弹出"首次运行设置"窗口（顶部黄色 banner）：

### Tab 1 · 拍照计划

计划由若干**条目**组成，每个条目独立配置：是否启用、模式（间隔 / 定时）、生效星期、以及时段或时间点。点 **+ 添加条目** 增加，每张卡片可单独删除。

默认自带「上午」「下午」两个**间隔条目**（周一~周五、每 30 秒一张）。按教室作息改即可，例如普通班级：

| 条目 | 模式 | 生效星期 | 时段 / 时间点 | 间隔 |
|---|---|---|---|---|
| 上午 | 间隔 | 周一~周五 | 08:00–11:30 | 30 秒 |
| 下午 | 间隔 | 周一~周五 | 13:30–17:00 | 30 秒 |

有早自习/晚自习，再 **+ 添加条目** 加间隔条目（如 07:00–07:50、18:30–21:30）。

**定时条目**用于"在精确时刻各拍一张"：把卡片模式切到「定时」，在时间点框里填逗号分隔的 HH:mm，例如打铃/升旗 `08:00, 08:55, 09:50, 14:00`。到点那一分钟内拍一张，不重复。

提示：
- **间隔** 30 秒/张是基线；班会、活动课需要更紧凑可调到 10 秒。
- 条目之间**允许时间重叠**（例如全天一个粗间隔条目，叠加某段更密的条目）。
- 底部「预计拍摄量」会按所有启用条目实时估算每周张数与磁盘占用。

### Tab 2 · 摄像头

下拉框列出**所有** USB 视频输入设备。希沃一体机典型会有：

- `USB Camera`（或 `HD Web Camera`）← 顶部内置摄像头，**通常选这个**
- `USB Document Camera`（或 `Honor Document Camera` 等）← **千万别选**，老师上课要用展台
- `Capture / Screen` 类 ← 也别选，是屏幕捕获设备

**勾选"使用设备最高分辨率"**（推荐）。如果发现画质过头电脑撑不住再去掉勾选手动填 1280×720。

点 **📷 用当前摄像头拍一张测试**：
- 成功会在屏幕上弹出图片查看器显示一张测试照片，验证摄像头能正常出图
- 失败可能是：① 选错了被希沃主软件占用的设备；② 设备驱动有问题

### Tab 3 · 存储

**输出文件夹**：建议 `D:\ClassLapse\Captures\`（同 D 盘但跟 .exe 分开）。**不要选 OneDrive 同步目录**，会被云盘频繁占用文件锁导致拍照失败。

**自动清理**：建议先**不开**，等用了一周看实际磁盘占用再回来调。

### Tab 4 · 关于

勾选 **开机自动启动 (HKCU\Run)** → 确定。这一步写注册表，无需管理员权限。

按顺序跑一遍，全部 ✅ 才能算部署成功。

## 4. 日常使用

| 想做的事 | 怎么做 |
|---|---|
| 临时不想拍（开会、检查等） | 右键托盘 → 暂停 1 小时 / 暂停今天剩余 |
| 恢复 | 右键托盘 → 恢复 |
| 改时段或摄像头 | 右键托盘 → 设置 |
| 看今天拍了几张 | 右键托盘看顶部状态（今日数量 + 最后成功拍摄时间） |
| 调试出问题 | 右键托盘 → 设置 → 关于 → 日志目录"打开" |

每次拍摄尝试还会写入 `%AppData%\ClassLapse\capture-journal\yyyy-MM-dd.jsonl`。这里会记录成功文件路径、分辨率、耗时，以及摄像头或写盘失败原因，适合排查某一天为什么出现断档。

## 5. 合成延时视频

拍完的照片可以在程序里直接合成 mp4，不必手动敲命令行。

### 先准备 ffmpeg（一次性）

本工具**不打包 ffmpeg**（避免发布体积翻倍）。三选一把 ffmpeg 提供给它：

- **放同目录**（最简单）：下载 Windows 版静态 `ffmpeg.exe`（gyan.dev 或 BtbN 的构建），放到 `D:\Tools\ClassLapse\`（与 `ClassLapse.exe` 同目录）。
- **装进 PATH**：把 ffmpeg 所在目录加进系统 PATH。
- **手动指定**：在合成窗口顶部点「选择 ffmpeg.exe」，选一次即记住。

> 用带 libx264 的构建（gyan.dev 的 essentials/full、BtbN 的 GPL 版都带）。若构建缺 libx264，程序会自动退回 mpeg4（能播但体积偏大、画质略低）并在窗口提示。

### 合成步骤

1. 右键托盘 → **🎬 合成延时视频...**
2. 顶部状态行确认已检测到 ffmpeg（否则按上面提供）。
3. 勾选要合成的日期——**可多选**，跨天/整学期会按时间顺序拼成一个长视频。
4. 选帧率（默认 30）与分辨率（默认 1080p；「原始」仅当所选日期分辨率一致时才安全）。
5. 确认输出路径（默认写进输出文件夹：单日 `2026-05-18.mp4`、多日 `2026-05-18_to_2026-06-30.mp4`）。
6. 点「开始合成」，进度条显示 `已合成 / 总帧`，可随时「取消」。
7. 完成后点「打开」在资源管理器里定位 mp4。

合成在后台进行，**不打断正常拍照**；整学期素材多时可能要几分钟（一体机算力有限，默认 `fast` 预设）。

> 帧率不对？极个别 ffmpeg 构建会忽略输入帧率（输出恒为 25fps）。这种情况打开配置文件把 `Timelapse.UseDurationListFallback` 设为 `true` 再合成。

## 6. FAQ

**Q：图标一直黄色不变绿**
A：摄像头被别的程序长期占用。日志会写明是哪种失败（DeviceBusy / Timeout）。打开希沃任务栏看是不是希沃的"摄像头"窗口没关。

**Q：开机后没看到托盘图标**
A：① 检查 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\ClassLapse` 是否存在；② 检查托盘是否被 Windows 折叠到了"^"展开按钮里；③ 看日志最近一条是否有 ERROR。

**Q：教室断电恢复后程序没起**
A：希沃白板的"硬关机"可能跳过用户 logon。在希沃后台启用 "Windows 自动登录"，开机自启就会触发。

**Q：希沃自带的"摄像头"应用打开能正常用吗？**
A：能。我们用的是独占式调用，一次只占设备 ~400ms。希沃应用打开的瞬间如果撞上我们的拍照，我们会失败一次（写 WARN 日志）；下次 tick 我们就让位（变黄、跳过）。老师关闭希沃相机后 5 分钟内我们再恢复。

**Q：怎么把照片合成延时视频？**
A：直接用内置功能——右键托盘 → 🎬 合成延时视频...（见上面第 5 节，可多选日期/整学期）。也可手动：进某个日期目录用 PowerShell 跑：
```powershell
ffmpeg -framerate 30 -pattern_type glob -i '*.jpg' -c:v libx264 -pix_fmt yuv420p out.mp4
```
或者把整个文件夹拖进剪映 → 自动按拍摄时间排序 → 设为相同短时长 → 导出。

**Q：能远程看实时画面吗？**
A：不行（也不打算做）。本工具是离线纯拍照，不开任何网络端口，避免安全审查。

## 7. 卸载

1. 右键托盘 → 退出
2. 右键托盘 → 设置 → 关于 → 取消勾选"开机自动启动"，点确定（写注册表删自启项）
3. 关掉设置窗口、关闭程序
4. 删除 `D:\Tools\ClassLapse\` 整个文件夹
5. （可选）删除 `%AppData%\ClassLapse\` 清理配置和日志
6. （可选）删除 `D:\ClassLapse\Captures\` 清理图片

注册表只动了 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\ClassLapse` 一个值，已经在步骤 2 删掉。无其他系统副作用。
